1. 전체 프로젝트 내 각각의 코드들이 일관된 패턴으로 정리.
     패턴을 분석해서 CRUD를 정리하고, 이를 준수해서 코드 작성.
2. One Source of Truth, 원칙 준수.
3. 적절하지 못하게 하드코딩한 값이 없도록 하기.
  중요한 값(Magic Numbers/Strings)을 코드에 직접 사용 – 취소, 완료와 같은 상태값 문자열이나 특정 숫자를 별도의 상수로 빼지 않고 코드에 그대로 박아 넣지 않도록 하기.
4. 에러 및 예외 처리에 대한 저가 정상적으로 사용하는 '해피 패스(Happy Path)' 케이스 뿐만 아니라 에러 처리나 예외 상황에 대한 대비가 되도록 작성.
5. Single Responsibility가 잘 구현되어 있는가?, 즉, 함수나모듈이  한가지  책임만 수행하도록 분리.  6. Shares파일/폴더의 관리가 잘 되어 있는가? 즉, 재사용성이 높은 함수나  컴포넌트를 모아놓아 코드의 확장성과 재사용성 보장.
